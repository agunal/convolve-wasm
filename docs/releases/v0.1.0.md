# @agunal/convolve-wasm v0.1.0

**Status:** Draft release notes. Do not publish, tag, or create a GitHub release until registry trusted-publishing controls, final merged-commit verification, and exact-tarball approval are complete.

## Overview

`@agunal/convolve-wasm` is a browser-side stereo audio convolution library backed by a Rust/WebAssembly DSP core and a dedicated module worker. Source audio and rendered output remain on the client.

## Highlights

- Public `CONVOLVE()` API for `.wav` and browser-decodable `.m4a` inputs.
- Full wet-only stereo linear convolution at 48 kHz.
- Optional beat detection from either input and equal-power alternating left/right panning.
- Optional exact sample-time reverse append with a complementary midpoint crossfade.
- Attenuation-only estimated true-peak normalization.
- Stereo 48 kHz signed 24-bit PCM WAV output.
- Dedicated module worker with transferable buffers and progress events.
- Exact 256 MiB conservative WASM allocation guard.
- Stable public error codes.
- Package-consumer tests proving that worker and standalone WASM assets remain local and bundle correctly.
- No bundled `@ffmpeg/core` dependency.

## Public API

```ts
import { CONVOLVE } from "@agunal/convolve-wasm";

const result = await CONVOLVE(
  { a: fileA, b: fileB },
  true,
  {
    beatPan: "a",
    panTransitionMs: 20,
    reverseCrossfadeMs: 5,
    targetDbtp: -1,
  },
);
```

```ts
CONVOLVE(
  audio: { a: File; b: File },
  appendReverse?: boolean,
  options?: ConvolveOptions,
): Promise<ConvolveResult>
```

## Browser and codec boundary

WAV is the automated portability baseline. M4A decoding uses the browser and operating system media stack through `decodeAudioData()` and may return `DECODE_FAILED` when the required codec is unavailable.

Automated WAV tests have passed in Chrome through `wasm-pack`, Playwright Chromium, and Playwright WebKit. The release HE-AAC matrix also passed against the unchanged production application in branded Chrome 149 on Windows Server 2025, branded Edge 149 on Windows Server 2025, and system Safari 26.4 on macOS 26.4. Every browser passed plain convolution, beat pan from A with reverse append, Blob playback, download-byte capture, browser-specific output-frame formulas, stereo 48 kHz PCM24 validation, finite non-silent peak checks, clipping checks, and page-error checks.

See [browser and codec support](../browser-support.md) and [the hosted release-browser evidence report](../testing/2026-07-15-hosted-release-browser-matrix.md) for exact versions, fixture hashes, output hashes, formulas, and artifact identities.

## Known limitations

- M4A support is browser/OS codec dependent rather than deterministic.
- Firefox M4A behavior is best effort and platform dependent.
- Processing is single-threaded WASM in a dedicated worker.
- Beat panning intentionally collapses the convolved stereo pair to one mono spatial object.
- v0.1.0 does not add dither.
- No server fallback or bundled FFmpeg decoder is included.
